# 팀원을 위한 Git 브랜치 생성 및 작업 가이드

이 문서는 팀원들이 개별 작업을 진행할 때, 메인 리포지토리(origin)와 동기화하고 개인 브랜치를 안전하게 생성하여 작업하는 방법을 설명합니다.

## 📌 1단계: 항상 최신 상태에서 시작하기

새로운 작업을 시작하거나 브랜치를 만들기 전에는 반드시 최신 `main` 코드를 내 컴퓨터로 받아와야 합니다. 
이를 통해 다른 팀원들의 작업물과 충돌(Conflict)이 발생하는 것을 최소화할 수 있습니다.

```bash
# 1. 메인 브랜치로 이동
git switch main

# 2. 원격 저장소의 최신 코드 받아오기
git pull origin main
```

## 📌 2단계: 내 전용 브랜치 만들기

규칙에 따라 `[본인이름]/[작업내용]` 형식으로 브랜치를 생성하고 이동합니다.

*   예시 (빛나 - LLM 파트 작업): `binna/feature/llm-guide`
*   예시 (효원 - 비전 API 작업): `hyowon/vision-api`

```bash
# 브랜치 생성과 동시에 이동 (-c 옵션)
git switch -c 본인이름/작업내용
```

## 📌 3단계: 원격 저장소(GitHub)에 내 브랜치 올리기

로컬(내 컴퓨터)에 새로 만든 브랜치를 팀원들이 볼 수 있도록 GitHub에 올려줍니다.
*최초 1회만 아래 명령어를 사용하며, 이후 변경 사항을 올릴 때는 `git push`만 입력하면 됩니다.*

```bash
git push -u origin 본인이름/작업내용
```

## 💡 브랜치 관련 유용한 명령어 모음

*   **현재 생성된 내 컴퓨터의 브랜치 목록 확인:**
    ```bash
    git branch
    ```
*   **원격 저장소(GitHub)를 포함한 모든 브랜치 목록 확인:**
    ```bash
    git branch -a
    ```
*   **작업 완료 후, 변경 사항 커밋 및 푸시하는 방법:**
    ```bash
    git add .
    git commit -m "작업 내용 설명 (예: LLM 가이드 파트 초안 작성)"
    git push
    ```
