# 팀원을 위한 Git 브랜치 생성 및 작업 가이드

이 문서는 팀원들이 개별 작업을 진행할 때, 메인 리포지토리(origin)와 동기화하고 개인 브랜치를 안전하게 생성하여 작업하는 방법을 설명합니다.

## 📌 1단계: 항상 최신 상태에서 시작하기

새로운 작업을 시작하거나 브랜치를 만들기 전에는 반드시 최신 `master` 코드를 내 컴퓨터로 받아와야 합니다. 
이를 통해 다른 팀원들의 작업물과 충돌(Conflict)이 발생하는 것을 최소화할 수 있습니다.

```bash
# 1. 메인 브랜치로 이동
git switch master

# 2. 원격 저장소의 최신 코드 받아오기
git pull origin master
```

## 📌 2단계: 내 전용 브랜치 만들기

규칙에 따라 `[본인이름]/[작업내용]` (예: `feature` 등의 단어 생략) 형식으로 짧게 브랜치를 생성하고 이동합니다.

*   예시 (빛나 - LLM 파트 작업): `binna/llm`
*   예시 (이연희 - OCR 파트 작업): `yeonhee/ocr`
*   예시 (운서 - 파트 작업): `woonseo/...` 
*   예시 (효원 - 비전 API 작업): `hyowon/vision-api`

```bash
# 브랜치 생성과 동시에 이동 (-c 옵션)
git switch -c 본인이름/작업내용
```

## 📌 3단계: 원격 저장소(GitHub)에 내 브랜치 올리기

로컬(내 컴퓨터)에 새로 만든 브랜치를 팀원들이 볼 수 있도록 GitHub에 올려줍니다.
*최초 1회만 아래 명령어를 사용하며, 이후 변경 사항을 올릴 때는 `git push`만 입력하면 됩니다.*

```bash
git push -u origin 본인이름/작업내용
```

## 💡 브랜치 관련 유용한 명령어 모음

*   **현재 생성된 내 컴퓨터의 브랜치 목록 확인:**
    ```bash
    git branch
    ```
*   **원격 저장소(GitHub)를 포함한 모든 브랜치 목록 확인:**
    ```bash
    git branch -a
    ```
*   **작업 완료 후 GitHub에 반영하기 (Commit & Push - 이연희님 예시):**
    
    작업을 마치고 코드를 안전하게 저장(Commit)한 뒤, 원격 저장소(GitHub)상단에 올리는(Push) 방법입니다. **연희님의 `yeonhee/ocr` 브랜치 작업 상황을 예시**로 단계별로 차근차근 설명합니다.

    1. **내 브랜치에 제대로 위치해 있는지 확인하기** (`yeonhee/ocr` 앞에 `*`가 있어야 합니다.)
        ```bash
        git branch
        ```
    2. **변경된 파일들을 모두 업데이트할 준비(Staging)를 합니다:**
        ```bash
        git add .
        ```
    3. **이번에 어떤 작업을 했는지 다른 팀원도 알기 쉽게 메모(메시지)를 남깁니다:**
        ```bash
        git commit -m "feat: OCR 정규식 파싱 등급 분류 로직 추가 및 DB 연동 준비"
        ```
    4. **마지막으로 내 컴퓨터의 작업 내역을 GitHub 원격 저장소의 내 브랜치 공간으로 밀어 올립니다:**
        ```bash
        git push origin yeonhee/ocr
        ```
