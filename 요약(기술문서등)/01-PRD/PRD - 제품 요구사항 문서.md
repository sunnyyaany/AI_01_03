## 1. 제품 개요

| 항목 | 내용 |
|------|------|
| **제품명** | 요약 (要藥) |
| **출시 플랫폼** | 토스 앱인토스 (Apps-in-Toss) |
| **제품 유형** | 웹뷰 기반 헬스케어 서비스 (Toss WebView) |
| **핵심 기술** | 비전 AI · OCR · LLM |
| **프론트엔드** | Vite + React + TypeScript + TDS |

### 제품 정의
비전 AI, OCR, LLM 기술을 융합하여 **알약 식별 + 처방전 분석 → 개인 맞춤형 복약 가이드** 를 제공하고, 약국과 환자를 직접 연결하여 맞춤형 복약 알림을 전송하는 서비스.

### 목표
- 환자의 복약 이해도 및 순응도를 획기적으로 높여 치료 효과 극대화
- 약국의 복약 지도 효율성 제고
- 환자 중심의 건강 관리 지원

## 2. 핵심 사용자 시나리오

> [!example] 시나리오 — 60대 당뇨 환자
> 약국 조제 후 약사로부터 **'요약' 앱으로 복약지도 푸시 알림** 수신
> → **실버 모드**(큰 글씨) + 자동 음성 안내 "혈당 조절 약, 식후 30분"
> → '다시 듣기' 버튼으로 반복 청취
> → AI 챗봇에 "이 약 먹으면 졸려요?" 질문
> → "졸음이 올 수 있으니 운전 시 주의하세요" 맞춤 답변 수신

## 3. 기능 요구사항

### P0 — MVP (필수)
→ [[P0 - MVP 핵심기능|상세 기능 명세 보기]]

| 기능 | 설명 |
|------|------|
| 알약 식별 (Vision AI) | GLM-5로 알약 촬영 → 약품 정보 자동 인식 |
| 복약지도문 OCR | 처방전·약 봉투 촬영 → 텍스트 추출 → 즉시 알림 생성 |
| LLM 맞춤형 가이드 | 의학 용어 쉽게 번역·요약, 실버 모드 UI, TTS 자동 재생 |
| AI 챗봇 (RAG) | 전문 지식베이스 기반 정확한 복약 Q&A |
| 복약 기록 대시보드 | 처방전·알약 이력 저장 및 관리 |

### P1 — Post-MVP (중요)
→ [[P1 - Post-MVP 기능|상세 기능 명세 보기]]

| 기능 | 설명 |
|------|------|
| 약국 연동 푸시 알림 | 약사 전용 웹 → 환자 앱으로 맞춤 복약 지도 발송 |
| 휴일 약국 찾기 | 네이버 플레이스 API 기반 위치 기반 운영 약국 안내 |
| 개인 건강 프로필 | 기저질환·알레르기 입력으로 정교한 맞춤 가이드 |
| 복약 순응도 트래킹 | 복용 체크 + 주간/월간 리포트 |

## 4. 비기능적 요구사항

| 항목 | 기준 |
|------|------|
| **성능** | AI 분석(OCR·Vision) → 가이드 생성 전체 **5초 이내** |
| **정확성** | OCR/Vision AI 인식률 **98% 이상** |
| **보안** | 전송: SSL/TLS, 저장: AES-256, PHI 암호화 |
| **사용성** | 실버/시니어 모드, Voice-First, 고대비·대형 폰트 |
| **플랫폼** | 토스 앱인토스 웹뷰 앱, 약사 전용 웹 관리 도구 |
| **디자인 시스템** | TDS (토스 디자인 시스템) 필수 사용 |
| **면책 고지** | 모든 정보 화면 하단 "의사/약사 상담을 대체할 수 없습니다" 상시 노출 |
| **LLM 안전성** | System Prompt 엔지니어링 + RAG 기반 환각 억제 |
