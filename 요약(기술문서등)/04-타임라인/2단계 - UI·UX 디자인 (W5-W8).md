# 2️⃣ 2단계 — UI·UX 디자인 (W5-W8)

> 상위 문서: [[전체 타임라인]] | [[🏠 요약 - 프로젝트 홈]]

---

## 체크리스트

### 🗺️ 사용자 여정 및 정보구조
- [ ] 사용자 여정 지도 (User Journey Map) 작성
  - 환자 여정: 약국 방문 → 앱 수신 → 복약 → 챗봇 상담
  - 약사 여정: 조제 완료 → 웹 접속 → 푸시 발송
- [ ] 정보 구조 (IA) 설계
- [ ] 화면 플로우 다이어그램 작성

### 📐 와이어프레임
- [ ] 모바일 앱 전체 화면 와이어프레임
  - 홈 / 카메라 촬영 / OCR 결과 / 복약 가이드 / 챗봇 / 이력 대시보드
- [ ] **실버 모드** 화면 별도 와이어프레임
- [ ] 약사 전용 웹 관리 도구 와이어프레임

### 🎨 UI 디자인 시안
- [ ] 앱인토스 브랜드 아이덴티티 계승한 디자인 가이드 수립
  - 컬러 팔레트, 타이포그래피, 아이콘 시스템
- [ ] 일반 모드 UI 디자인 시안 (고해상도)
- [ ] **실버/시니어 모드** UI 디자인 시안
  - 폰트 크기 기본 대비 150% 이상
  - 고대비 색상 (WCAG AA 기준)
  - 핵심 정보만 노출 (약 이름·복용 시간·횟수)
- [ ] 약사 전용 웹 UI 시안

### 🔊 음성 인터페이스 설계
- [ ] TTS 자동 재생 UX 플로우 설계
- [ ] '다시 듣기' 버튼 위치 및 인터랙션 설계
- [ ] 재생 속도·음량 조절 UI 설계

### 🧪 사용성 테스트
- [ ] 타겟 사용자 모집 (고령층 3-5명, 약사 2-3명)
- [ ] 프로토타입 기반 UT 시나리오 작성
- [ ] 사용성 테스트 진행
- [ ] 피드백 수집 및 디자인 반영

---

## 디자인 원칙

| 원칙 | 설명 |
|------|------|
| **단순함** | 앱 실행 → 촬영 → 확인, 3단계 이내 핵심 가치 경험 |
| **가독성** | 고대비 색상, 대형 폰트, Voice-First |
| **직관성** | 기능 설명 최소화, 보편적 아이콘 활용 |
| **브랜드 계승** | 앱인토스 디자인 요소 일관성 유지 |
| **신뢰성** | AI 정보임 명시, 출처 표기, 경고 강조 |

---

## 주차별 계획

| 주차 | 핵심 목표 |
|------|-----------|
| **W5** | 사용자 여정 지도, IA, 화면 플로우 확정 |
| **W6** | 와이어프레임 완성 (앱 + 웹) |
| **W7** | UI 디자인 시안 제작 (일반 + 실버 모드) |
| **W8** | 사용성 테스트 진행 → 피드백 반영 → 최종 시안 확정 |
