# 4️⃣ 4단계 — 테스트 및 QA (W21-W24)

> 상위 문서: [[전체 타임라인]] | [[🏠 요약 - 프로젝트 홈]]

---

## 체크리스트

### 🔬 알파 테스트 (내부)
- [ ] 내부 직원 대상 알파 테스트 계획 수립
- [ ] 테스트 시나리오 작성 (핵심 기능 P0 전체)
- [ ] 알파 테스트 진행
- [ ] 버그 수집 및 우선순위 분류

### ⚙️ 기능 테스트
- [ ] 알약 식별 정확도 검증 (목표 98% 이상)
- [ ] OCR 텍스트 추출 정확도 검증 (목표 98% 이상)
- [ ] LLM 복약 가이드 품질 검토 (의약 전문가 리뷰)
- [ ] TTS 음성 품질 및 재생 정확성 확인
- [ ] 실버 모드 UI 접근성 검증 (WCAG AA)
- [ ] 복용 알림 정확도 테스트
- [ ] 챗봇 응답 품질 검토 (RAG 정확성)
- [ ] 약사 웹 → 환자 앱 푸시 알림 E2E 테스트

### 🚀 성능 테스트
- [ ] AI 분석 → 가이드 생성 전체 프로세스 5초 이내 확인
- [ ] 동시 접속자 부하 테스트
- [ ] API 응답 시간 측정

### 📱 호환성 테스트
- [ ] iOS 주요 버전 테스트 (최소 iOS 15 이상)
- [ ] Android 주요 버전 테스트 (최소 Android 10 이상)
- [ ] 다양한 기기 해상도 테스트 (특히 실버 모드)

### 🔒 보안 테스트
- [ ] 취약점 점검 (OWASP Mobile Top 10)
- [ ] 개인정보 암호화 확인 (전송·저장)
- [ ] 접근 제어 로직 검증
- [ ] PHI 데이터 유출 시뮬레이션 테스트

### 👥 클로즈드 베타 테스트
- [ ] 외부 사용자 그룹 모집 (고령층, 약사 포함)
- [ ] 베타 테스트 진행 (1-2주)
- [ ] 사용자 피드백 수집 및 분석
- [ ] 핵심 피드백 반영 및 수정

### 🐛 버그 수정 및 안정화
- [ ] P0 버그 전체 수정
- [ ] P1 버그 우선순위 기반 처리
- [ ] 최종 성능 확인
- [ ] 앱스토어 제출 전 최종 점검 (앱스토어 가이드라인 준수)
