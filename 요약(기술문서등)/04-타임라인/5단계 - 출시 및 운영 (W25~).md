# 5️⃣ 5단계 — 출시 및 운영 (W25~)

## 출시 체크리스트

### 📱 앱스토어 심사 제출
- [ ] **Google Play Store** 심사 제출
  - [ ] 앱 설명, 스크린샷, 개인정보처리방침 URL 준비
  - [ ] 헬스케어 앱 관련 추가 정책 확인 및 준수
- [ ] **Apple App Store** 심사 제출
  - [ ] 앱 심사 가이드라인 4.x (의료·건강 카테고리) 검토
  - [ ] 개인정보처리방침, 이용약관 준비
- [ ] 앱인토스 업데이트 버전 배포

### 🚀 MVP 공식 출시
- [ ] 출시 공지 및 마케팅 자료 준비
- [ ] 약국 파트너 출시 안내 (약사 웹 사용법 포함)
- [ ] 출시 기념 이벤트 (선택)

## 운영 체크리스트 (상시)

### 📊 모니터링
- [ ] 사용자 피드백 채널 운영 (앱 리뷰, CS 채널)
- [ ] 서버 오류·장애 모니터링 대시보드 구축
- [ ] AI 인식 오류 리포트 모니터링

### 📈 KPI 트래킹

| KPI | 목표 | 측정 방법 |
|-----|------|-----------|
| 누적 다운로드 수 | — | Play/App Store Console |
| 월간 활성 사용자 (MAU) | — | 앱 분석 도구 |
| AI 인식 성공률 | 98% 이상 | 서버 로그 |
| 복약 가이드 생성 수 | — | DB 집계 |
| 챗봇 상담 만족도 | — | 인앱 평가 |
| 앱 평점 | 4.0 이상 | 앱스토어 |
| 약국 연동 수 (P1) | — | 약사 웹 가입 수 |

### 🔄 지속적 개선
- [ ] 월별 사용자 피드백 분석 리포트 작성
- [ ] RAG 지식베이스 월 1회 정기 업데이트
- [ ] 프롬프트 엔지니어링 지속 최적화
- [ ] AI 모델 성능 점검 및 개선

### 🗺️ 차기 버전 로드맵
- [ ] P1 기능 개발 계획 수립 (약국 연동 푸시, 건강 프로필, 순응도 트래킹)
- [ ] 출시 후 3개월 내 P1 기능 출시 목표 설정
