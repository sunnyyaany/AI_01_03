## 1. 소개

### 1.1 문서 목적
본 문서는 "요약" 프로젝트의 소프트웨어 요구사항을 정의합니다. 복약 관리 AI 서비스의 기능적/비기능적 요구사항을 명세하여, 개발팀과 이해관계자 간의 명확한 의사소통을 지원합니다.

### 1.2 프로젝트 범위
- **프로젝트 명**: 요약 - AI 기반 복약 관리 서비스
- **목적**: 알약 식별, 처방전 OCR, 개인 맞춤형 복약 가이드 제공
- **대상 사용자**: 일반 사용자, 고령층(실버/시니어), 보호자, 약사
- **플랫폼**: 토스 앱인토스 웹뷰 기반 웹 애플리케이션

### 1.3 용어 정의

| 용어 | 정의 |
|------|------|
| PHI | Personal Health Information, 개인건강정보 |
| OCR | Optical Character Recognition, 광학 문자 인식 |
| Vision AI | 이미지 기반 객체 인식 인공지능 |
| RAG | Retrieval-Augmented Generation, 검색 증강 생성 |
| TTS | Text-To-Speech, 텍스트 음성 변환 |
| JWT | JSON Web Token, 인증 토큰 방식 |
| OAuth 2.0 | 소셜 로그인을 위한 표준 인증 프로토콜 |
| TDS | Toss Design System, 토스 디자인 시스템 |
| MSA | Microservices Architecture, 마이크로서비스 아키텍처 |

### 1.4 참조 문서
- IEEE 830: IEEE Recommended Practice for Software Requirements Specifications
- NIA 요구사항 정의서 작성 가이드
- 개인정보보호법
- 의료법 AI 서비스 규정

## 2. 전체 시스템 설명

### 2.1 시스템 개요
본 시스템은 AI 기술을 활용하여 사용자의 복약 관리를 돕는 헬스케어 서비스입니다. YOLOv8 기반 알약 인식, Naver Clova OCR 기반 처방전 분석, GPT 기반 개인 맞춤형 복약 가이드 생성 기능을 제공합니다.

### 2.2 시스템 아키텍처
- **Frontend**: React/Vue.js (토스 앱인토스 웹뷰)
- **Backend**: FastAPI (MSA 기반)
- **AI/ML**: YOLOv8s, Naver Clova OCR, OpenAI GPT-4o-mini/4o
- **Database**: 관계형 DB + Vector DB (Pinecone/FAISS)
- **Storage**: AWS S3
- **Push Notification**: 토스 앱인토스 알림 API

### 2.3 주요 기능
1. 회원 관리 및 인증 (JWT)
2. 건강 프로필 관리
3. 알약 식별 (Vision AI)
4. 처방전 OCR 및 관리
5. 개인 맞춤형 복약 가이드 생성
6. AI 챗봇 상담
7. 복약 알림 및 순응도 추적
8. 약사 연동 (P1)
9. 약국 찾기 (P1)
10. 가족 관리 (P2)
11. 병용금기 체크 (P2)

## 3. 기능 요구사항

## 3.1 회원관리

### REQ-AUTH-001: 회원가입
- **요구사항 상세 설명**: 모든 비로그인 유저는 회원가입을 진행할 수 있습니다.
  - 회원가입에 필요한 항목
    - 이메일
    - 비밀번호
    - 이름
    - 연령
    - 성별
- **기능/비기능**: 기능
- **우선순위**: 높음
- **비고**: JWT 기반 인증
- **상태**: 대기중

### REQ-AUTH-002: 로그인
- **요구사항 상세 설명**: 로그인 유저는 이메일과 비밀번호를 입력하여 로그인할 수 있습니다.
  - 입력 항목
    - 이메일
    - 비밀번호
- **기능/비기능**: 기능
- **우선순위**: 높음
- **비고**: JWT 토큰 발급 (24시간 유효)
- **상태**: 대기중

### REQ-AUTH-003: 토큰 갱신
- **요구사항 상세 설명**: 로그인 유저는 Refresh Token을 사용하여 Access Token을 갱신할 수 있습니다.
- **기능/비기능**: 기능
- **우선순위**: 높음
- **비고**: Refresh Token 유효기간: 30일
- **상태**: 대기중

### REQ-AUTH-004: 로그아웃
- **요구사항 상세 설명**: 로그인 유저는 로그아웃을 진행할 수 있습니다.
- **기능/비기능**: 기능
- **우선순위**: 중간
- **상태**: 대기중

### REQ-AUTH-005: 구글 소셜 로그인
- **요구사항 상세 설명**: 모든 비로그인 유저는 구글 계정으로 간편 로그인/회원가입을 진행할 수 있습니다.
  - 인증 흐름
    - 구글 로그인 버튼 클릭
    - 구글 OAuth 2.0 인증 페이지로 리다이렉트
    - 사용자 동의 후 콜백
    - 자동 회원가입 또는 로그인 처리
  - 수집 정보
    - 이메일
    - 이름
    - 프로필 사진 (선택)
- **기능/비기능**: 기능
- **우선순위**: 높음
- **비고**: OAuth 2.0 표준 프로토콜, JWT 토큰 발급
- **상태**: 대기중

### REQ-AUTH-006: 카카오 소셜 로그인
- **요구사항 상세 설명**: 모든 비로그인 유저는 카카오 계정으로 간편 로그인/회원가입을 진행할 수 있습니다.
  - 인증 흐름
    - 카카오 로그인 버튼 클릭
    - 카카오 OAuth 2.0 인증 페이지로 리다이렉트
    - 사용자 동의 후 콜백
    - 자동 회원가입 또는 로그인 처리
  - 수집 정보
    - 이메일 (선택 동의)
    - 닉네임
    - 프로필 사진 (선택)
- **기능/비기능**: 기능
- **우선순위**: 높음
- **비고**: OAuth 2.0 표준 프로토콜, JWT 토큰 발급, 국내 사용자 우선 지원
- **상태**: 대기중

## 3. 알약 식별

### REQ-VISION-001: 이미지 촬영
- **요구사항 상세 설명**: 모든 유저는 웹뷰 카메라를 통해 알약 이미지를 촬영하거나 갤러리에서 선택할 수 있습니다.
  - 입력
    - 웹뷰 카메라 촬영 (HTML5 getUserMedia)
    - 또는 파일 선택
- **기능/비기능**: 기능
- **우선순위**: 높음
- **비고**: 카메라 접근 권한 처리, 파일 업로드 인터페이스 제공
- **상태**: 대기중

### REQ-VISION-002: 알약 인식
- **요구사항 상세 설명**: 촬영된 알약 이미지를 Vision AI를 통해 자동 인식합니다.
  - 처리
    - YOLOv8s 기반 객체 탐지
    - FastAPI 메타데이터 재랭킹 (색상/모양/각인)
  - 출력
    - 약품 이름
    - 성분
    - 분류 정보
- **기능/비기능**: 기능
- **우선순위**: 높음
- **비고**: 정확도 목표: 98% 이상, 5초 이내 결과 표시, AIhub 경구약제 이미지 데이터 활용
- **상태**: 대기중

### REQ-VISION-003: 인식 결과 표시
- **요구사항 상세 설명**: 알약 인식 결과를 사용자에게 표시합니다.
  - 표시 항목
    - 약품명
    - 성분
    - 효능
    - 인식 신뢰도
- **기능/비기능**: 기능
- **우선순위**: 높음
- **비고**: 인식 실패 시 "다시 촬영해주세요" 안내
- **상태**: 대기중

### REQ-VISION-004: 인식 이력 조회

## 6. 복약 가이드

### REQ-GUIDE-001: 가이드 생성
- **요구사항 상세 설명**: 알약 식별 또는 처방전 OCR 결과를 바탕으로 개인 맞춤형 복약 가이드를 생성합니다.
  - 입력
    - OCR/Vision 추출 데이터
    - 사용자 건강 프로필
  - 처리
    - OpenAI API (GPT-4o-mini/4o) + RAG
  - 출력
    - 쉬운 언어의 복약 가이드
    - 복용 방법
    - 주의사항
    - 생활습관 가이드
- **기능/비기능**: 기능
- **우선순위**: 높음

- **상태**: 대기중

### REQ-GUIDE-002: 의학 용어 번역
- **요구사항 상세 설명**: 복잡한 의학 용어를 쉬운 언어로 번역하여 표시합니다.
- **기능/비기능**: 기능
- **우선순위**: 높음
- **비고**: LLM을 통한 자동 번역
- **상태**: 대기중

### REQ-GUIDE-003: 실버/시니어 모드
- **요구사항 상세 설명**: 고령층을 위한 실버 모드 UI를 제공합니다.
  - 실버 모드 사양
    - 폰트 크기: 일반 대비 150% 이상 (24px 이상)
    - 표시 정보: 약 이름 · 복용 시간 · 횟수만 간결하게
    - 색상 대비: WCAG AA 기준 이상
    - 활성화: 설정 또는 자동(연령 기반)
- **기능/비기능**: 기능
- **우선순위**: 높음
- **비고**: TDS 디자인 시스템 활용, 반응형 레이아웃
- **상태**: 대기중

### REQ-GUIDE-004: TTS 자동 음성 안내
- **요구사항 상세 설명**: 복약 가이드 생성 완료 시 음성으로 자동 재생합니다.
  - TTS 엔진
    - Web Speech API (브라우저 네이티브)
    - 또는 백엔드 TTS 서비스
  - 기능
    - 자동 재생
    - 다시 듣기 버튼

### 5.4 통신 인터페이스

#### 5.4.1 프로토콜
- **HTTP/HTTPS**: RESTful API 통신
- **TLS 1.2 이상**: 모든 데이터 전송 암호화
- **WebSocket** (선택): 실시간 챗봇 응답 (P2)

#### 5.4.2 데이터 형식
- **Request/Response**: JSON
- **이미지 업로드**: multipart/form-data
- **문자 인코딩**: UTF-8

#### 5.4.3 인증
- **JWT (JSON Web Token)**: 사용자 인증
- **Access Token**: 24시간 유효
- **Refresh Token**: 30일 유효
- **Header**: `Authorization: Bearer {token}`
